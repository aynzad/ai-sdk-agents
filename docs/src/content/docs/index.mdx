## Overview

**ai-sdk-agents** is a thin, zero-runtime-dependency TypeScript library that adds multi-agent orchestration on top of Vercel AI SDK's primitives (`generateText`, `streamText`, `tool()`). It does not replace AI SDK — it composes on top of it.

```bash
pnpm add @aynzad/ai-sdk-agents ai zod
```

```ts
import { Agent, Runner } from "@aynzad/ai-sdk-agents";
import { openai } from "@ai-sdk/openai";

const agent = new Agent({
  name: "Assistant",
  model: openai("gpt-4o"),
  instructions: "You are a helpful assistant.",
});

const result = await Runner.run(agent, "Write a haiku about recursion.");
console.log(result.output);
```

## Why AI SDK Agents?

Vercel AI SDK gives you powerful primitives. OpenAI's Agents SDK gives you multi-agent orchestration. **ai-sdk-agents** bridges the gap — adding handoffs, guardrails, and tracing on top of AI SDK without locking you into a framework. [Read more about where it fits in the ecosystem](/ai-sdk-agents/guides/why/).

## Core Primitives

The SDK has a small set of primitives:

- **Agents** — LLMs equipped with instructions and tools
- **Handoffs** — transfer full conversation control between agents
- **Guardrails** — validate inputs and outputs with tripwire-based halting, plus [8 built-in presets](/ai-sdk-agents/guides/guardrails/#guardrail-presets) for jailbreak detection, content moderation, PII filtering, and more
- **Runner** — orchestration loop that manages the full agent execution
- **Tracing** — structured observability for every agent run
